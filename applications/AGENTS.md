# Applications

## Purpose

The deployable NukeIDE shells: the Theia browser application, the Electron desktop application, and the all-in-one Docker deployment.

## Ownership

All files under `applications/` except generated artifacts (`*/lib/`, `*/node_modules/`, `*/src-gen/`, `gen-webpack*.js`, `*.tsbuildinfo`, `electron/dist/`, `docker/workspace/`).

## Local Contracts

- `browser/` and `electron/` are Theia application packages. Their `package.json` dependency lists decide which extensions ship â€” registering a new extension means adding it to **both**.
- `src-gen/`, `gen-webpack.config.js`, `gen-webpack.node.config.js`, and `lib/` are produced by the Theia generator/webpack during builds. Never edit them; regenerate instead.
- `browser/` scripts: `bundle` (rebuild + `theia build`), `start`, `watch`. Driven from the repo root via `yarn build:browser` / `yarn start:browser` / `yarn watch:browser`.
- `electron/` scripts: `bundle` / `bundle:prod` (rebuild â†’ preload generation â†’ `theia build` â†’ `patch-electron-main`), `start`, `dist` (electron-builder). Driven from the root via `yarn build:electron(:prod)`, `yarn start:electron`, `yarn dist`, `yarn package:electron(:all)`.
- `electron/src-gen/backend/electron-main.js` is patched after every bundle by `scripts/patch-electron-main.js` (sets `THEIA_DEFAULT_PLUGINS` so bundled VS Code extensions load in packaged apps); the bundle scripts already run it.
- `resources/preload.html` (repo root `resources/`) is generated by `scripts/generate-preload.js` from `resources/preload.template.html` with version + tips injected; edit the template or `resources/tips.yml`, never the output.
- `docker/` builds the all-in-one container: stage 1 compiles the browser app, stage 2 (miniforge) provides the `ide` conda env defined in `environment.yml` (Python 3.13, Node 22, OpenMC, Trame/ParaView/VTK, gmsh/CadQuery/pythonocc-core, pydagmc/moab, headless VTK). `compose.yml` mounts `docker/workspace/` as `/root/workspace`.

## Work Guidance

- Adding or removing an extension: update both app `package.json` files, then re-bundle. Extension code changes alone only need the extension's `build` plus an app re-bundle.
- Electron packaging runs on CI for Linux/Windows/macOS (`.github/workflows/build.yml`); local packaging is `yarn package:electron`.
- The Docker image targets the browser app only â€” do not wire Electron into it.
- Runtime Python dependencies for extension backends belong in `docker/environment.yml` (and `docs/installation.md`), not in the root `requirements-dev.txt`, which is dev/test-only.

## Verification

```bash
yarn build:browser      # webpack bundle of the browser app
yarn build:electron     # webpack bundle of the Electron app
yarn docker:build       # container image
```
